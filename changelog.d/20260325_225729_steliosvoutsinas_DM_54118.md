### Added

- PostgreSQL backend support, allowing a single Docker image to serve QServ, BigQuery, or PostgreSQL backends selected via the BACKEND environment variable.
- Added PostgresAdqlQueryImpl, ObsCoreRegionConverter, PostgresFormatFactory, PostgresQueryRunner classes for the PostgreSQL backend.
- Added PluginFactory-pg.properties: plugin configuration for the PostgreSQL backend.
- Added jdbc/tapschemauser JNDI datasource in context.xml for pg backend tap_schema reads.

### Changed

- Restructured packages by domain concern: query/ (ADQL translation), runner/ (execution), schema/, storage/, upload/, validation/, util/.
- Renamed dialect/ package to query/ to better reflect its purpose as the ADQL translation layer.
- Renamed RubinQueryRunner to KafkaQueryRunner and moved to runner/ package alongside new PostgresQueryRunner.
- KafkaContextListener skips Kafka initialisation when BACKEND=pg.
- QueryJobManager routes to ThreadPoolExecutor (pg) or KafkaJobExecutor (qserv/bigquery) based on BACKEND env var.
- Moved web layer classes to the ws package.
- Updated all three PluginFactory*.properties files to reflect new package structure.
- Updated web.xml servlet mappings to reflect new ws package.
