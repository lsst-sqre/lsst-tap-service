### Added

- PostgreSQL backend support, allowing a single Docker image to serve QServ, BigQuery, or PostgreSQL backends selected via the BACKEND environment variable.
- Added PostgresAdqlQueryImpl ObsCoreRegionConverter, PostgresFormatFactory, PostgresQueryRunner classes
- Added PluginFactory-pg.properties`: plugin configuration for the PostgreSQL backend.
- Add jdbc/tapschemauser JNDI datasource in context.xml for pg backend tap_schema reads.

### Changed

- QueryJobManager routes to ThreadPoolExecutor(pg) or KafkaJobExecutor(qserv/bigquery) based on BACKEND env var.
- Moved Kafka execution infrastructure to the kafka package.
- Moved web layer classes to the ws package.
- Renamed RubinQueryRunner to KafkaQueryRunner and moved to impl/runner/.
- Moved ALMATapSchemaDAO from schema/ into impl/ alongside the other plugin implementations.
- Updated web.xml servlet to reflect new ws package.
- Updated all three PluginFactory*.properties files with new ALMATapSchemaDAO class.
